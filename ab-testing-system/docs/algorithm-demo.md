## 🧮 核心算法公式

### 偏差计算公式

```java
/**
 * 计算分组偏差
 * 
 * deviation = targetRatio - actualRatio
 * 
 * 其中：
 * - targetRatio：目标比例（固定）
 * - actualRatio：实际比例 = count / total
 * 
 * 偏差为正：该组分配不足，优先分配
 * 偏差为负：该组分配过多，减少分配
 */

for (int i = 0; i < 4; i++) {
    Group group = experiment.getGroup(i);
    
    double targetRatio = group.getTargetRatio();
    // A组：0.10
    // B组：0.20
    // C组：0.30
    // D组：0.40
    
    double actualRatio = (double) globalCounters[i].get() / total;
    // 例如：A组实际 = 1 / 10 = 0.10 (10%)
    
    double deviation = targetRatio - actualRatio;
    // 例如：A组偏差 = 0.10 - 0.10 = 0
    
    if (deviation > maxDeviation) {
        maxDeviation = deviation;
        selectedGroup = group.getName();
    }
}
```

### 分配策略

```
if (用户在缓存中) {
    返回缓存的分组（100%一致）
} else {
    计算每个组的偏差
    选择偏差最大的组（贪心策略）
    缓存分组结果
}
```

## 🎯 关键优势

### 1. 100%用户一致性

**原理**：
```
第1次访问：user_001 -> 计算分配 -> A组 -> 缓存
第2次访问：user_001 -> 缓存命中 -> A组
第3次访问：user_001 -> 缓存命中 -> A组
...
永远：A组
```

### 2. 完美分组精度

**原理**：
```
动态调整，每次选择偏差最大的组

初始：
A:10% B:20% C:30% D:40%

第1个用户：
A:100% B:0% C:0% D:0% → 分配D组（偏差最大）

第2个用户：
A:50% B:0% C:0% D:50% → 分配C组（偏差最大）

...

第10个用户：
A:10% B:20% C:30% D:40% → 完美！
```

### 3. 适应任何用户分布

**地域性活动**：
```
90%广东用户，用户ID: gd_user_001, gd_user_002, ...

Hybrid算法：
- 不依赖用户ID的hash值
- 只根据当前偏差分配
- 结果：A:10% B:20% C:30% D:40%（完美）

AdaptiveHash：
- 依赖hash值分布
- gd_user_001 ~ gd_user_100 的hash可能集中在某个区间
- 结果：A:5% B:45% C:30% D:20%（失衡）
```

## 📈 性能分析

### 时间复杂度

```
缓存命中（老用户）：
- 查询缓存：O(1)
- 返回结果：O(1)
- 总计：O(1)

缓存未命中（新用户）：
- 查询缓存：O(1)
- 计算偏差：O(4) = O(1)
- 缓存结果：O(1)
- 更新计数器：O(1)
- 总计：O(1)
```

### 空间复杂度

```
用户缓存：O(N)
- N = 用户数量
- 每个用户存储一个字符串（分组名）
- 100万用户 ≈ 100MB内存

计数器：O(1)
- 4个分组计数器
- 固定大小
```
