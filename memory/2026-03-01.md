# 2026-03-01 预研笔记

## A/B 实验动态分流系统

### 核心挑战
- 1000万+ 用户规模
- 5维标签 × 10值 = 10^5 组合单元
- QPS 10,000+，延迟 < 50ms
- 实时分布对齐（各组标签分布 = 大盘分布）
- 粘性分流（同一用户结果不变）

### 方案对比

| 维度 | 方案一：贪心计数 | 方案二：自适应哈希 |
|------|-----------------|-------------------|
| 一致性 | 强一致 | 最终一致 |
| 性能 | 锁竞争严重 | 热路径无锁 |
| 复杂度 | 低 | 中 |
| 适用场景 | 低 QPS | 高 QPS |

### 技术选型
- Java 17（Record、Var、Virtual Thread）
- Spring Boot 3.x
- Caffeine（本地缓存）
- HdrHistogram（延迟统计）
- React + TailwindCSS + ECharts（前端）

### 模拟策略
- **Redis 锁竞争**：分段锁（Striped Lock）
- **HBase 延迟**：LockSupport.parkNanos()
- **热点分布**：Zipf Distribution
- **流量偏置**：前 N 秒只推某个标签的用户
