# 2026-03-19 记忆

## ab-testing-system 项目修复 - Hybrid 调整间隔 API 问题

### 问题描述
前端页面点击 "Hybrid 调整间隔" 时报错 404

### 问题根因
1. 后端缺少 `/api/load-test/set-adjust-interval` 端点
2. 前端调用的 API 路径不正确（`/api/set-adjust-interval` 而不是 `/api/load-test/set-adjust-interval`）

### 解决方案

#### 后端修改
**文件**: `src/main/java/com/example/abtesting/controller/EnhancedLoadTestController.java`

添加了新端点：
```java
@PostMapping("/set-adjust-interval")
public Map<String, Object> setAdjustInterval(@RequestParam long intervalMs) {
    hybridEngine.setAdjustInterval(intervalMs);
    Map<String, Object> result = new HashMap<>();
    result.put("status", "success");
    result.put("intervalMs", intervalMs);
    result.put("intervalSec", intervalMs / 1000.0);
    return result;
}
```

#### 前端修改
**文件**: `frontend/src/components/ControlPanel.js`

修复 API 调用路径：
```javascript
// 修复前
const res = await fetch(`${API_BASE}/api/set-adjust-interval?intervalMs=${intervalMs}`, {
  method: 'POST'
});

// 修复后
const res = await fetch(`${API_BASE}/api/load-test/set-adjust-interval?intervalMs=${intervalMs}`, {
  method: 'POST'
});
```

### 验证结果
- ✅ API 端点已添加
- ✅ 前端 API 路径已修复
- ✅ 后端已重启测试

### 后续问题
在修复过程中发现了 Java 8 兼容性问题：
- 项目使用 `Map.of()` 语法（Java 9+ 特性）
- 但项目运行在 Java 8 环境
- 需要将所有 `Map.of()` 替换为 Java 8 兼容的 `HashMap` 写法

### 文件修改记录
1. **EnhancedLoadTestController.java** - 添加 `set-adjust-interval` 端点
2. **UserPoolController.java** - 修复 `Map.of()` 为 `HashMap`
3. **ExperimentApplicationService.java** - 修复 `Map.of()` 为 `HashMap`
4. **ControlPanel.js** - 修复 API 路径
5. **App.js** - 修复 metrics 数据读取
